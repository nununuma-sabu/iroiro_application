# ピンクノイズ生成画面 (Mock)

このリポジトリは、**ピンクノイズ生成用のWeb画面モック**です。  
現在は UI 部分のみを実装しており、実際のピンクノイズ生成処理やファイル出力処理は未実装です。

---

## 概要

- ブラウザ上でピンクノイズ生成条件を入力するためのフォーム画面
- 「再生時間」「音量」「サンプリングレート」「チャンネル数」「出力ファイル名」を指定できる構成:contentReference[oaicite:0]{index=0}  
- 将来的に、バックエンド(API)と連携して WAV ファイルなどを生成・ダウンロードすることを想定したモック

---

## ファイル構成

- `index.html`  
  - 入力フォームとレイアウトを定義したメインHTML:contentReference[oaicite:1]{index=1}
- `style.css`  
  - 全体のレイアウト、フォーム、スライダー、ボタン等のスタイル定義:contentReference[oaicite:2]{index=2}
- `script.js`  
  - 音量スライダーの値表示更新  
  - フォーム送信時の処理（現状はデータ収集のみでAPI呼び出しは未実装）:contentReference[oaicite:3]{index=3}

## 動作要件

- モダンブラウザ（Chrome / Edge / Firefox / Safari など）
- ローカルファイルを開ける環境  
  （※サーバーは不要で、`index.html` をブラウザで直接開くだけで動作します）

## セットアップ（インストール）

1. このリポジトリを取得します。
   - ZIP でダウンロードする or Git で clone するなど、お好きな方法で構いません。
2. 取得したフォルダ内の `index.html` をブラウザで開きます。

ビルド工程はなく、追加のライブラリも使用していないため、そのまま動作します。

---

## 使い方

1. ブラウザで `index.html` を開きます。
2. フォームに各項目を入力します。:contentReference[oaicite:4]{index=4}  
   - **再生時間 (秒)**: 例 `600` など、生成したいピンクノイズの長さ（秒）  
   - **音量**: スライダーで 0.0〜1.0 の範囲を指定  
     - ラベル右側に現在値が小数第2位まで表示されます（例: `0.30`）:contentReference[oaicite:5]{index=5}  
   - **サンプリングレート (Hz)**: 例 `44100` など  
   - **チャンネル数**: ラジオボタンでステレオ (2ch) / モノラル (1ch) を選択  
   - **出力ファイル名**: 例 `pink_noise.wav`
3. フォーム下部の **「ピンクノイズ生成」ボタン**を押します。  
   - 現在は、入力された項目を JavaScript で取得するところまでのモック実装となっています。

## 実装のポイント

### HTMLフォーム構成

- `form` 要素の中に、以下の入力が配置されています。:contentReference[oaicite:6]{index=6}  
  - `input#duration` (テキスト): 再生時間（秒）
  - `input#volume` (range): 音量（0.0〜1.0、0.01刻み）
  - `input#samplerate` (テキスト): サンプリングレート
  - `input[name="channels"]` (radio): ステレオ/モノラル選択
  - `input#filename` (テキスト): 出力ファイル名
  - `button.generate-button`: 送信ボタン

### JavaScript (`script.js`)

- DOMContentLoaded イベントで初期化処理を実行:contentReference[oaicite:7]{index=7}  
  - 音量スライダー (`#volume`) と表示用要素 (`#volume-display`) を取得
  - スライダーの値変更時 (`input` イベント) に表示値を更新
    - 値は `toFixed(2)` で小数第2位までに整形
- フォーム送信時 (`submit` イベント) に `e.preventDefault()` を行い、画面遷移を抑止
  - `FormData` → `Object.fromEntries()` でフォーム内容をオブジェクト化
  - 現在はコンソール出力やAPI呼び出しはコメントアウト状態（プレースホルダー）

### スタイル (`style.css`)

- 画面中央寄せのレイアウトやコンテナの影・角丸など、簡易なカード風デザイン:contentReference[oaicite:8]{index=8}  
- タイトル部分に背景色を付与して目立たせる
- 音量ラベル (`#volume-label-group`) は `display: flex` でラベルと値を左右に配置
- スライダーとラジオボタンはブラウザ標準の見た目からカスタマイズし、青系のテーマカラーに統一

## 注意事項（モックとして）

- **本リポジトリはUIモックであり、実際のピンクノイズ生成や音声再生は行いません。**
- 音量や再生時間などの値は、今後の実装のための「想定パラメータ」に過ぎません。
- 実際にピンクノイズを再生・利用する際は、聴覚への影響に十分注意し、安全な音量で使用する必要があります。
- 本モックで定義している値の範囲やデフォルト値は、あくまでサンプルです。実プロダクトで使用する場合は、要件に合わせて適宜調整してください。
